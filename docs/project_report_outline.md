# Каркас отчета по проекту

## 1. Актуальность

- Рост объема цифровых исторических источников
- Необходимость быстро извлекать место, время и тип события
- Польза визуальной аналитики для учебных проектов и исследований

## 2. Постановка задачи

- Принимать исторические тексты разных форматов
- Извлекать события в структурированном JSON
- Нормализовать даты и топонимы
- Геокодировать события
- Показывать события на карте, в таблице и на timeline
- Добавить этап ручной верификации

## 3. Архитектура

- Обоснование выбора FastAPI, React, PostgreSQL, PostGIS
- Разделение на ingest, LLM, dates, geocoding, repositories, frontend
- Поток данных от файла до карты

## 4. Реализация

- Backend API
- Схема БД
- LLM-пайплайн и mock режим
- Нормализация дат
- Геокодинг и кэш
- React frontend

## 5. Результаты

- Поддержка загрузки нескольких форматов
- Автоматическое извлечение событий
- Фильтрация и отображение на карте
- Экспорт CSV/JSON/GeoJSON
- Human-in-the-loop проверка

## 6. Ограничения

- Ограниченность mock extractor
- Неполный охват исторических дат и вариантов топонимов
- Ограниченность локального gazetteer
- Отсутствие асинхронного фонового execution pipeline

## 7. Перспективы

- Подключение Mordecai 3 / HeidelTime / MAVEN
- Расширение исторического geocoding cache
- Фоновые задачи и очереди
- Сравнение нескольких документов и построение кампаний
- Более глубокая аналитика и spatial queries
